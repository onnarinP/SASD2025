# HW 1: เลือกฟังคลิปแล้วสรุปเนื้อหา
Link: https://youtu.be/guycIP56YeY  
ช่วงนาทีที่ 23:52 – 27:36  
สรุป:  
Kent Beck กับ Dave Farley คุยกันเรื่องความหมายที่แท้จริงของ TDD (Test-Driven Development)และทำไมมันถึงยังสำคัญอยู่ Kent เริ่มจากการโต้แย้งมุมมองที่บอกว่า TDD ไม่ใช่เทคนิคการทดสอบ
โดยเขายืนยันชัดเจนว่า TDD คือเทคนิคการทดสอบแน่นอน เพราะสุดท้ายมันได้ชุดเทสที่บอกได้ว่าผ่านหรือไม่ผ่าน แต่ในขณะเดียวกัน เขาก็ย้ำว่า TDD ไม่ได้มีค่าแค่เรื่องเทสอย่างเดียว
Kent มองว่า TDD เป็นทั้งเทคนิคการออกแบบ เพราะมันให้ feedback เร็วมากเกี่ยวกับ API และตรรกะของระบบ เป็นเทคนิคการกำหนดสเปก เพราะบังคับให้เราต้องเขียนสมมติฐานของระบบออกมาให้ชัด
และที่เขาให้ความสำคัญมากที่สุดเป็นเหมือน “Xanax ของโปรแกรมเมอร์” ที่ช่วยลดความเครียดและความกังวลเวลาเขียนโค้ด นอกจากนี้ TDD ยังช่วยให้การทำงานร่วมกันในทีมง่ายขึ้นด้วย
เขายังบอกด้วยว่าเขาไม่ได้จะมาบังคับให้ใครทำ TDD ไม่ทำก็ได้ แต่ไม่ว่าจะทำหรือไม่ทำ ทุกคนก็ยังต้องเจอปัญหาเดิม ๆ อยู่ดี ทั้งเรื่องการทดสอบ การออกแบบ การอธิบายพฤติกรรมของระบบ การทำงานเป็นทีม และการจัดการความไม่มั่นใจของตัวเอง 
ซึ่ง TDD เป็นแค่เครื่องมือหนึ่งที่ช่วยให้เรารับมือกับปัญหาเหล่านี้ได้เป็นระบบมากขึ้น ช่วงท้าย Dave พูดถึงการแยกพฤติกรรมออกจากผลลัพธ์ แต่ Kent ช่วยปรับให้ชัดว่าแก่นจริง ๆ คือการแยก “พฤติกรรมของระบบ” ออกจาก “โครงสร้างของโค้ด” 
พฤติกรรมคือสิ่งที่ผู้ใช้สนใจ ส่วนโครงสร้างคือการจัดโค้ดภายใน การแยกสองอย่างนี้ออกจากกันทำให้เราปรับปรุงหรือ refactor โค้ดได้อิสระ โดยไม่ไปกระทบสิ่งที่ระบบควรทำ ซึ่งเป็นแนวคิดที่ Kent เชื่อมั่นอย่างมากในตอนนี้  
### คำศัพท์เพิ่มเติม  
- ***Test-Driven Development (TDD)*** คือ แนวทางการพัฒนาซอฟต์แวร์ที่เริ่มจากการเขียน เทสก่อนเขียนโค้ดจริง โดยใช้เทสเป็นตัวกำหนดว่าระบบควรทำอะไร แล้วจึงเขียนโค้ดให้น้อยที่สุดเพื่อทำให้เทสนั้นผ่าน จากนั้นค่อยปรับปรุงโครงสร้างโค้ดให้ดีขึ้น
- ***Xanax*** คือ ชื่อทางการค้าของยา Alprazolam ซึ่งเป็นยากลุ่ม benzodiazepine ใช้สำหรับ
  - ลดความวิตกกังวล
  - บรรเทาอาการแพนิค
  - ช่วยให้รู้สึกผ่อนคลายและสงบลงอย่างรวดเร็ว
- ***Specification technique*** คือ เทคนิคในการกำหนดหรืออธิบายพฤติกรรมของระบบอย่างชัดเจน ว่าระบบ 'ควรทำอะไร' โดยไม่เน้นว่า 'ทำอย่างไร'
- ***Design technique*** คือ เทคนิคที่ช่วยในการออกแบบซอฟต์แวร์ ไม่ใช่แค่วิธีเขียนโค้ดให้ทำงานได้ แต่ช่วยให้เราตัดสินใจได้ว่าโค้ด ควรถูกออกแบบอย่างไร
